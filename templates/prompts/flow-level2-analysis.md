## 分析规则

1. **条件分支必须完整**：每个 if-else、switch-case、三元表达式都要体现在流程图中
2. **异常必须标注**：每个 Asserts.check / throw 都要列出，标明触发条件
3. **事务边界必须清晰**：标注 @Transactional 的方法、异步执行的代码块、finally 中的操作
4. **Redis 锁必须标注**：tryLock 的 key 构成、超时时间、获取失败的处理
5. **外部系统调用必须标注**：HTTP 调用、MQ 发送、缓存操作
6. **状态变更必须追踪**：setStatus / updateStatus 调用，标明从什么状态到什么状态
7. **代码优先**：所有信息从代码中提取，不要猜测或编造
8. **入参从 Java DTO 推断 TS 格式**：字段名保持 camelCase，类型映射（String→string, Integer→number, List→array）
